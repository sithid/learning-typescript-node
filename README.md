# learning-typescript-node

A barebones Node.js command-line app for learning TypeScript — no frameworks, no bundlers, just the language.

## Setup

```bash
npm install
```

## Run

**During development** (runs TypeScript directly, no build step):

```bash
npm run dev
npm run dev -- Alice
```

**Production-style** (compile first, then run JavaScript):

```bash
npm run build
npm start
npm start -- Alice
```

## Project layout

```
src/
  main.ts        ← start here; add new .ts files as you learn
dist/            ← compiled JavaScript (generated by npm run build)
```

## Suggested learning path

Work through these concepts by editing `src/main.ts` or adding new files under `src/`:

1. **Basic types** — `string`, `number`, `boolean`, arrays, `null` / `undefined`
2. **Functions** — parameter types, return types, optional params (`name?: string`)
3. **Interfaces & type aliases** — shape objects with `interface User { ... }`
4. **Unions & literals** — `type Status = "pending" | "done"`
5. **Generics** — `function first<T>(items: T[]): T | undefined`
6. **Classes** — `class Todo { ... }`
7. **Modules** — split code into separate files and import/export
8. **Async** — `async`/`await` with `fetch` or `fs/promises`
9. **CLI patterns** — read `process.argv`, parse flags, read stdin

## TypeScript config

`tsconfig.json` has **strict mode** enabled. This catches more mistakes early — ideal for learning. If a rule feels too noisy at first, comment it out in `tsconfig.json` and re-enable it later.

## Useful commands

| Command | What it does |
|---------|--------------|
| `npm run dev` | Run `src/main.ts` with hot reload via tsx |
| `npm run build` | Compile TypeScript → JavaScript in `dist/` |
| `npm start` | Run the compiled output |
| `npx tsc --noEmit` | Type-check without writing files |
